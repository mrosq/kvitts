# 015 – PWA (installerbar, offline-kapabel)

**Status:** open
**Skapad:** 2026-04-24

## Varför

Kvitts används typiskt under resor — alltså ofta på mobilen, ibland utan
pålitligt nät (flyg, utlandsroaming, dålig täckning på en restaurang). All
data ligger redan i localStorage, så appen är *funktionellt* offline-klar.
Det som saknas är:

1. Att kunna lägga den på hemskärmen som en riktig app (utan browserchrome).
2. Att kunna öppna den utan nät efter första besöket.

PWA löser båda utan backend, utan app store, utan kostnad.

## Vad

Tre tillägg till repot:

**`manifest.json`** i roten:
- `name`: "Kvitts", `short_name`: "Kvitts"
- `start_url`: "/", `display`: "standalone"
- `theme_color` + `background_color` som matchar nuvarande UI
- `icons`: minst 192×192 och 512×512 PNG (+ en "maskable" variant för Android)

**`sw.js`** i roten — enkel cache-first service worker:
- Vid `install`: pre-cacha `index.html`, `app.js`, `logic.js`, `manifest.json`, ikonerna
- Vid `fetch`: serva från cache, fall tillbaka till nätverk
- Vid `activate`: rensa gamla cache-versioner
- Versionskonstant överst (`CACHE_NAME = "kvitts-v1"`) — bumpas manuellt vid
  deploys som ändrar assets, annars servas gammal version för evigt

**Ändringar i `index.html`:**
- `<link rel="manifest" href="manifest.json">`
- `<link rel="apple-touch-icon" href="icon-192.png">` (iOS läser inte manifest fullt ut)
- `<meta name="theme-color" content="...">`

**Ändring i `app.js`:**
- `navigator.serviceWorker.register('sw.js')` vid load, bakom feature-check

## Öppna frågor / noteringar

- **Ikon-design**: behövs en enkel Kvitts-ikon. Textlogga på solid bakgrund
  räcker för v1 — kan förfinas senare. Avgör: skapas manuellt eller via
  någon generator (t.ex. realfavicongenerator)?
- **Cache-busting vid deploy**: manuell bump av `CACHE_NAME` är enklast men
  lätt att glömma. Alternativ: inkludera en build-hash, men det kräver
  byggsteg vilket vi medvetet undviker. Manuell bump först, se om det blir
  problem.
- **"Single-file lokalt"-ambitionen**: CLAUDE.md säger att `index.html` ska
  kunna öppnas som fil. Det är redan delvis brutet (separata JS-filer).
  Service worker kräver att appen serveras över http(s) — SW:n registreras
  inte från `file://`. Appen fungerar fortfarande som vanlig sida från fil,
  bara PWA-lagret aktiveras inte. Uppdatera CLAUDE.md så förväntan
  överensstämmer med verkligheten: primärt körsätt är Vercel eller lokal
  http-server.
- **iOS-begränsningar**: Safari stödjer PWA men med restriktioner
  (localStorage kan rensas om appen är oanvänd länge — "7-day rule" för
  appar som inte lagts till hemskärmen). För Kvitts räcker det, men värt
  att veta.
- **Avgränsning**: ingen push-notiser, ingen background sync, ingen
  install-prompt-knapp i UI:t i v1. Webbläsaren sköter prompten själv.

## Att göra som del av leveransen

- Uppdatera CLAUDE.md: "öppna som fil"-förväntan skrivs om — primärt körsätt
  är Vercel eller lokal http-server. Single-file-ambitionen dokumenteras
  som historisk.
- Dokumentera cache-busting-rutinen (bump `CACHE_NAME` i `sw.js` vid
  asset-ändringar) på ett ställe som framtida-jag faktiskt läser — förslag:
  kort rad i CLAUDE.md under "Deploy".
- **Ikon**: enkel "K" på solid bakgrund räcker för v1. Avgörs under
  implementation.
